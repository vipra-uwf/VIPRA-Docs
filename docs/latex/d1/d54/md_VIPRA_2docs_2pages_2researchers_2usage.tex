The following are requirement for running the VIPRA simulation.


\begin{DoxyEnumerate}
\item \href{https://cmake.org/download/}{\texttt{ CMake}} version 3.\+14 or newer
\item A compiler that supports C++20
\item Linux
\end{DoxyEnumerate}

Additionally, for distributed simulations\+:


\begin{DoxyEnumerate}
\item MPI, currently \href{https://www.open-mpi.org/software/ompi/v5.0/}{\texttt{ open\+MPI}} is known to work
\end{DoxyEnumerate}\hypertarget{md_VIPRA_2docs_2pages_2researchers_2usage_autotoc_md763}{}\doxysection{\texorpdfstring{Creating A Simulation}{Creating A Simulation}}\label{md_VIPRA_2docs_2pages_2researchers_2usage_autotoc_md763}

\begin{DoxyItemize}
\item {\ttfamily -\/map=\texorpdfstring{$<$}{<}path\texorpdfstring{$>$}{>}}\+: path to the map file (required)
\item {\ttfamily -\/peds=\texorpdfstring{$<$}{<}path\texorpdfstring{$>$}{>}}\+: path to the pedestrian initial positions
\item {\ttfamily -\/params=\texorpdfstring{$<$}{<}path\texorpdfstring{$>$}{>}}\+: path to the parameter file, default = ./module\+\_\+params.json
\item {\ttfamily -\/modules=\texorpdfstring{$<$}{<}path\texorpdfstring{$>$}{>}}\+: path to the sim config file, default = ./sim\+\_\+config.json
\item {\ttfamily -\/count=\texorpdfstring{$<$}{<}number\texorpdfstring{$>$}{>}}\+: number of simulations to run, default = 1 (more on this in Parameter Sweeps)
\item {\ttfamily -\/install=\texorpdfstring{$<$}{<}path\texorpdfstring{$>$}{>}}\+: path to module install directory, default = ./install
\item {\ttfamily -\/log=\texorpdfstring{$<$}{<}debug\texorpdfstring{$\vert$}{|}info\texorpdfstring{$\vert$}{|}warn\texorpdfstring{$\vert$}{|}error\texorpdfstring{$>$}{>}}\+: minimum level to log, default = info

Parameters are loaded into the simulation from a JSON file, each module implementation has its own set of parameters.

An example set of parameters is provided in the {\ttfamily examples/module\+\_\+params.\+json} file.\hypertarget{md_VIPRA_2docs_2pages_2researchers_2usage_autotoc_md767}{}\doxysubsection{\texorpdfstring{Simulation Config}{Simulation Config}}\label{md_VIPRA_2docs_2pages_2researchers_2usage_autotoc_md767}
A simulation config file is a JSON file that tells VIPRA which modules to use.

The types of modules are discussed in\+: \doxysectlink{md_VIPRA_2docs_2pages_2researchers_2modules}{Modules}{0}

An example simulation config is provided in the {\ttfamily examples/sim\+\_\+config.\+json} file.\hypertarget{md_VIPRA_2docs_2pages_2researchers_2usage_autotoc_md768}{}\doxysubsection{\texorpdfstring{Parameter Sweeps}{Parameter Sweeps}}\label{md_VIPRA_2docs_2pages_2researchers_2usage_autotoc_md768}
Parameters can be randomized, in parallel or serial, to get a wide range of possible outcomes.

A map defineds the geometry and special areas used by the simulator. These maps include obstacles, spawn positions, objectives for agents to use as goals, and special areas that change agent behavior.
