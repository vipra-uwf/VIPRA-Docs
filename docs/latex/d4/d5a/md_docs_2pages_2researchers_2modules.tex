The VIPRA framework is designed to be customizable to individual needs. This is done through {\ttfamily Modules}, each module is responsible for different parts of a simulation.

\doxysectlink{md_docs_2pages_2developers_2overview}{Developers can extend VIPRA through {\ttfamily Modules}}{0}.\hypertarget{md_docs_2pages_2researchers_2modules_autotoc_md347}{}\doxysubsection{\texorpdfstring{Module Types}{Module Types}}\label{md_docs_2pages_2researchers_2modules_autotoc_md347}
Modules can be categorized into 6 different types.
\begin{DoxyItemize}
\item {\ttfamily Model}
\item {\ttfamily Goals}
\item {\ttfamily Map}
\item {\ttfamily Pedestrian Set}
\item {\ttfamily Input}
\item {\ttfamily Output}
\end{DoxyItemize}

Some \doxysectlink{md_docs_2pages_2researchers_2base__modules}{Base Module Implementations}{0} are provided with VIPRA

\DoxyHorRuler{0}
\hypertarget{md_docs_2pages_2researchers_2modules_autotoc_md349}{}\doxysubsubsection{\texorpdfstring{Model}{Model}}\label{md_docs_2pages_2researchers_2modules_autotoc_md349}
{\ttfamily Model} modules are the pedestrian dynamics models that govern how the pedestrians will move during a simulation.

\DoxyHorRuler{0}
\hypertarget{md_docs_2pages_2researchers_2modules_autotoc_md351}{}\doxysubsubsection{\texorpdfstring{Goals}{Goals}}\label{md_docs_2pages_2researchers_2modules_autotoc_md351}
Goals modules handle pathfinding for the simulation.

\DoxyHorRuler{0}
\hypertarget{md_docs_2pages_2researchers_2modules_autotoc_md353}{}\doxysubsubsection{\texorpdfstring{Map}{Map}}\label{md_docs_2pages_2researchers_2modules_autotoc_md353}
Map modules hold the geometry of the environment and handle map/pedestrian collision detection.

The main focus of Map modules is performance.

\DoxyHorRuler{0}
\hypertarget{md_docs_2pages_2researchers_2modules_autotoc_md355}{}\doxysubsubsection{\texorpdfstring{Pedestrian Set}{Pedestrian Set}}\label{md_docs_2pages_2researchers_2modules_autotoc_md355}
Pedestrian Set modules hold the pedestrian positions/velocities and handle queries to pedestrians (ex. nearest neighbor).

The main focus of the pedestrian set is performance, but can also be used to add constraints to how pedestrians can move (i.\+e. celluar automata vs continuous fields)

\DoxyHorRuler{0}
\hypertarget{md_docs_2pages_2researchers_2modules_autotoc_md357}{}\doxysubsubsection{\texorpdfstring{Input}{Input}}\label{md_docs_2pages_2researchers_2modules_autotoc_md357}
Input modules handle loading input for the simulation.

These include the\+:
\begin{DoxyItemize}
\item {\ttfamily Map Input}\+: responsible for loading in the map file
\item {\ttfamily Pedestrian Input}\+: responsible for loading in initial pedestrian positions
\end{DoxyItemize}

\DoxyHorRuler{0}
\hypertarget{md_docs_2pages_2researchers_2modules_autotoc_md359}{}\doxysubsubsection{\texorpdfstring{Output}{Output}}\label{md_docs_2pages_2researchers_2modules_autotoc_md359}
Output modules handle producing output from the simulation.

Multiple can be added to one simulation to produce.
